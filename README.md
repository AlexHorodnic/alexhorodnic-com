## Frontend Approach

My frontend work is guided by practical product quality: clear UX, responsive layouts, reusable components, and code that can be maintained after the first version ships.

I care about:

- Component-driven Angular development
- Clean visual hierarchy and readable interfaces
- Maintainable SCSS and consistent design tokens
- Realistic product screens instead of isolated demos
- Polished loading, hover, focus, and interaction states
- Accessibility, keyboard usability, and mobile behavior

## Responsive Design

The portfolio is built with responsive behavior as a core requirement, not an afterthought.

Responsive details include:

- Mobile-first layout improvements
- Touch-friendly spacing and controls

- Optimized preview layouts for screenshots and generated project visuals

## Local Development

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm start
```

Open:

```text
http://localhost:4200
```

Build for production:

```bash
npm run build
```

Run tests:

```bash
npm test
```
