Product Context

Private by architecture, not by promise

Most AI workspaces send prompts, files, and conversation context to a hosted model. That creates a privacy boundary users cannot inspect and a recurring inference cost the frontend does not control.

Aether tests a different product decision: move inference and retrieval onto the user's device. There is no application AI endpoint, no exposed API key, and no document upload path. The tradeoff is that model quality and speed depend on the user's browser and GPU.

System Design

A local pipeline with visible boundaries

The product separates workspace state, document retrieval, and model execution so each layer can fail clearly without blocking the whole interface.

Key Challenge

Grounding must be inspectable

A source selector is dangerous if the interface implies that an answer used documents when retrieval actually found nothing.

The decision

Every assistant message stores an explicit state: grounded, general, or no match. Grounded answers expose named citations. A failed match is labeled as a general answer instead of quietly borrowing credibility from the selected collection.

Model reasoning is removed by a streaming state machine before output can reach rendering, conversation history, or the clipboard.

Product Resilience

Local AI needs honest failure states

Device-aware models

Four profiles explain speed and memory tradeoffs instead of promising equal quality.

Automatic fallback

A failed larger model releases its worker and retries with the cached Fast profile.

Controllable generation

Streaming responses support Stop, Retry, Regenerate, and reasoning-safe Copy actions.

Deliberate settings

Model, identity, theme, and response changes use explicit Save and Discard behavior.

Tradeoffs

A focused local product, not a hosted-model replacement

Aether deliberately avoids online search, shared accounts, and server-side sync. Those features need secured gateways, permissions, provider contracts, and monitoring; adding them directly to the frontend would weaken the privacy model.

Retrieval uses lightweight lexical ranking rather than local embeddings, and small browser models remain less capable than leading hosted systems. WebGPU support, download size, speed, and available memory also vary by device.
